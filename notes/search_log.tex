\section{Purpose}
This document logs the best Quantum Tanner codes found by our search scripts.

For each group (and local-code choice), we want to record:
\begin{itemize}
  \item \textbf{Quantum code:} $(n,k)$ and best distance upper bound $d_{\mathrm{ub}}$ returned by QDistRnd, plus the number of trials / steps used.
  \item \textbf{Construction data:} the group $G$, generator multisets $A$ and $B$, and the permutations used for the local codes.
  \item \textbf{Classical slice Tanner codes:} the 4 associated classical Tanner codes (two A-slice codes and two B-slice codes), with their parameters and distance-estimation settings.
  \item \textbf{Artifacts:} paths to saved parity-check matrices for the quantum code and for the 4 classical codes.
\end{itemize}

\section{Folder convention for best-code artifacts}
All matrices + metadata for the best codes should be stored under:
\[
\texttt{best\_codes/}
\]
with one subfolder per saved code (or per ``milestone'').

Recommended structure:
\begin{itemize}
  \item \texttt{best\_codes/meta/<CODE\_TAG>.json} (all metadata: group, A/B, perms, trials, etc.)
  \item \texttt{best\_codes/matrices/<CODE\_TAG>\_HX.mtx}
  \item \texttt{best\_codes/matrices/<CODE\_TAG>\_HZ.mtx}
  \item \texttt{best\_codes/matrices/<CODE\_TAG>\_classical\_A\_H0.mtx}, etc. (4 classical matrices)
\end{itemize}

\noindent \textbf{Naming suggestion for \texttt{<CODE\_TAG>}:}\\
\texttt{G=<group>\_n=<n>\_k=<k>\_dub=<dub>\_<timestamp>\_A=<A\_id>\_B=<B\_id>}\\
(avoid spaces; keep it filesystem-friendly).
